### 브랜치란?

git을 사용하는 목적은 다수와 함께 사용하는 것이다. 한 커밋 버전 기준으로 두명이 

동시에 수정하게 된다면??  이것을 해결하기 위한 개념이 브랜치이다. 

마스터 브랜치는 커밋 1,2,3 등 고정될 수 있다. 다른 브랜치1 이 커밋 4를 기준으로 하여

각  커밋을 나눈 브랜치 각각에서 작업을 수행할 수 있으며 커밋은 분산될 수 있다.

이 각각의 브랜치를 왔다갔다 하는 것이 헤드이다.

협업자는 커밋을 올릴 각각의 브랜치를 만든다.

자신이 만든 브랜치로 이동한 후 브랜치에 커밋을 올린다. 코딩이 완료되면 브랜치를

합치는 과정이 들어간다.

[Master]브랜치에는 직접 커밋을 올리지 않는다. 기능 개발을 하기 전에 마스터 브랜치

를 기준으로 새로운 브랜치를 만든다. 각 브랜치에서 작업을 마친 후 합치는 것임.

각 Feature와 속성에 관련되게 만드는 것..

#### 실습

소스트리에서 어떠한 브랜치 아래에 만들기, 체크아웃설정 후 헤드가 넘나들게 한다.

두 브랜치 사이을 넘나드는 각 과정이 필요하다. 즉 헤드를 변경해야 함

브랜치 목록에 마스터 브랜치 한 후 체크 아웃을 해주면 해당 브랜치가 마스터를 가리

키게 된다.



### 브랜치 병합

병합에는 여러가지 유형이 있다. 

Merge commit  병합시 각 브랜치의 특징을 모두 갖고 있는 것

Fast-forward 병합시 한 브랜치의 특성과 같게 되는 것(병합이 가능하지만 새로운 병합

커밋이 생기지는 않는다.)  

 Conflict 병합시 한 특성에 대한 두개의 변수가 있어서 커밋 시에 충돌이 난다.

병합시 나오는 뒤 숫자는 커밋의 두단계 더 앞서 있다는 뜻이다.

마스터 브랜치는 여러 사용자와 함께 하는 기둥이다.
